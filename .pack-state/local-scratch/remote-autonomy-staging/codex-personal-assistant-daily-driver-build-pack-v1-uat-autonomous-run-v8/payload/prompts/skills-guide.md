# Skills Guide

The initial assistant skill catalog is intentionally small.

- `identity_surface`: explain who this assistant is and what it is for
- `operator_alignment`: explain what the operator is optimizing for and how
  the assistant should keep that in view
- `partnership_policy`: explain how the assistant should handle ambiguity,
  drift, and accountability
- `context_router`: map a request to the smallest relevant local surfaces
- `restart_memory`: preserve cross-session continuity
- `workspace_bootstrap`: export a portable preview bundle
- `doctor`: check local health before deeper work

Customization overlays should stay local and additive.
