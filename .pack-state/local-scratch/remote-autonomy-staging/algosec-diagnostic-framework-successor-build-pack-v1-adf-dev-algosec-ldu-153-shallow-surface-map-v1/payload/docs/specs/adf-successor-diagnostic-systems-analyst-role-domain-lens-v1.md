# ADF Successor Diagnostic Systems Analyst Role/Domain Lens v1

## Purpose

Define the active initial role/domain lens for the current ADF successor build
pack.

## Lens Name

`diagnostic-systems-analyst`

## Active Meaning In This Build Pack

Use this lens to frame the current work as:

- bounded live system mapping first
- machine-readable diagnostic structure first
- support-useful stop points before deeper subsystem theory

For the current first wave, this means:

- start from observable Rocky 8 host and appliance surfaces
- map processes, services, ports, configs, logs, and JVM visibility
- distinguish fact from inference explicitly
- stop once one useful shallow surface map and one reviewable summary exist

## Guardrails

Do not use this lens to justify:

- broad architecture reconstruction before evidence exists
- complete dependency mapping in the first slice
- predictive analytics in the first slice
- large playbook or publication work before the shallow surface map is accepted
